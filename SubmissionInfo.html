Abstract

The Semantic Automated Discovery and Integration (SADI) design patterns intend to guide the creation of service-oriented architectures (not necessarily Web-centric) that natively consume and produce Linked Data, and are therefore more intereropable with other Linked Data/Semantic Web resources. As it is only aimed at defining service behavior, SADI is orthogonal to other Semantic Web Service technologies and standards such as OWL-S or SAWSDL. We also provide a reference implementation showing how SADI has been applied to the creation of Semantic Web Services. This implementation has undergone multiple successful scholarly peer-reviews, and has been enacted by multiple institutions worldwide, including by individuals/organizations outside of the group of submitting Members.
